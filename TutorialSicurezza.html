Tecniche di Preprocessing per Reti Neurali in Sicurezza Informatica

1. Indirizzi IP

Gli IP possono essere rappresentati come numeri normalizzati:

  • Ogni ottetto (0-255) viene diviso per 255 → otteniamo valori tra 0 e 1.
  • Esempio: 192.168.0.1 → [0.752, 0.658, 0, 0.0039]

Questa rappresentazione è compatta e pronta per l'input della rete neurale.

2. Porte

Le porte possono essere rappresentate in due modi:

  • Numerico normalizzato: Dividi la porta per 65535 (numero massimo di porta) → ottieni valori continui tra 0 e 1.
  • One-hot: Se vuoi distinguere solo alcune porte importanti (80, 443, 22), puoi usare un vettore binario:
80 → [1,0,0], 443 → [0,1,0], 22 → [0,0,1]

3. Protocolli

I protocolli (TCP, UDP, ICMP, ecc.) sono categorie discrete e limitate, quindi conviene usare il one-hot encoding:

  • TCP → [1,0,0]
  • UDP → [0,1,0]
  • ICMP → [0,0,1]

4. Link e URL

Per link e URL, ci sono due approcci principali:

  • ASCII normalizzato: Trasformi ogni carattere in numero con ord() e dividi per 255.
  • One-hot (più adatto per testi lunghi): Ogni carattere o parola diventa un vettore binario basato sul vocabolario.

Esempio ASCII: https://sito.it → [0.709, 0.721, 0.721,...]

Esempio one-hot: si crea un vocabolario dei caratteri o delle parole e ogni posizione del vettore rappresenta un carattere/parola presente o assente.

5. Email

Per le email brevi conviene usare l'ASCII normalizzato:

  • Ogni carattere dell’email diventa un numero tra 0 e 1.
  • Target: 0 = legittima, 1 = sospetta/malware

Per il corpo dell’email o testi lunghi si può usare one-hot sulle parole o tecniche di embedding.

6. File binari (Virus/Malware)

Per i file binari si usa la cosiddetta sequenza binaria:

  • Ogni byte del file viene letto come numero tra 0 e 255.
  • Si normalizza dividendo per 255 → valori tra 0 e 1.
  • Questi numeri diventano input della rete neurale per rilevare malware o virus.
  • Se il file è molto grande, si possono prendere solo i primi N byte o segmentare in blocchi.

Esempio: byte_data = [31, 160, 195] → normalized = [0.122, 0.627, 0.765]

7. Sintesi dei metodi

  • IP → numeri normalizzati
  • Porte → numeri normalizzati o one-hot (solo poche porte importanti)
  • Protocolli → one-hot
  • Link/email brevi → ASCII normalizzato
  • Link/email lunghi → one-hot parole o embedding
  • File binari/virus → sequenza binaria normalizzata

Tutti questi dati possono essere poi combinati in un vettore unico come input per una rete neurale che rileva attacchi, malware, phishing o file sospetti.
